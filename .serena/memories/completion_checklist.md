When finishing work in this project, prefer to run checks proportional to the touched area. For `sure_copy_core`, at minimum run `cargo test -p sure_copy_core`; if API or persistence code changed, also consider `cargo test --workspace`, `cargo fmt --all`, and `cargo clippy --workspace --all-targets -- -D warnings`. If frontend code changed, run `npm run build` and any relevant UI tests. Avoid reverting unrelated user changes because the worktree may already be dirty.
